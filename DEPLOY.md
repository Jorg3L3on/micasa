# Deployment Guide — Vercel + Neon

## Overview

This app uses:
- **Vercel** for hosting (Next.js App Router)
- **Neon** for PostgreSQL (serverless-compatible driver in production)

---

## Step 1 — Set up Neon

1. Create an account at [neon.tech](https://neon.tech)
2. Click **New Project**, choose a name and a region close to your Vercel deployment (e.g. `us-east-1`)
3. Once created, go to **Connection Details**
4. Select **Pooled connection** and copy the connection string — it looks like:
   ```
   postgresql://user:pass@ep-xxx-pooler.us-east-1.aws.neon.tech/dbname?sslmode=require
   ```

---

## Step 2 — Run migrations against Neon

Set the Neon connection string in your local `.env.local`:

```env
DATABASE_URL="postgresql://user:pass@ep-xxx-pooler.us-east-1.aws.neon.tech/dbname?sslmode=require"
```

Then apply all existing migrations (do **not** use `migrate dev` — that is for local only):

```bash
npx prisma migrate deploy
```

Optionally seed the database:

```bash
npx prisma db seed
```

After this, revert `DATABASE_URL` in `.env.local` back to your local Postgres string if you want to continue developing locally.

---

## Step 3 — Deploy to Vercel

### 3.1 — Import the project

1. Go to [vercel.com](https://vercel.com) → **Add New Project**
2. Import your GitHub repository
3. Vercel will auto-detect Next.js

### 3.2 — Override the build command

In **Settings → General → Build & Development Settings**, set the Build Command to:

```
prisma generate && next build
```

### 3.3 — Set environment variables

In **Settings → Environment Variables**, add the following:

| Variable | Value |
|---|---|
| `DATABASE_URL` | Neon pooled connection string from Step 1 |
| `AUTH_SECRET` | Your secret key (see `.env.local` — the value already generated by `npx auth`) |
| `AUTH_URL` | Your Vercel deployment URL, e.g. `https://your-app.vercel.app` |

### 3.4 — Deploy

Click **Deploy**. Vercel will run `prisma generate && next build` and publish the app.

---

## Environment variables reference

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `AUTH_SECRET` | Yes | JWT signing secret for NextAuth v5 |
| `AUTH_URL` | Yes | Base URL of the deployed app |

---

## Notes

- **File uploads** are capped at **4MB** (Vercel serverless limit). This applies to receipt PDFs and credit card statement imports.
- **Cold starts**: Neon free tier scales to zero when idle. The first request after a period of inactivity may take ~1 second longer while the database wakes up.
- **Local development** continues to use your local PostgreSQL instance via `pg.Pool`. The Neon serverless driver is only used when `NODE_ENV=production`.
- **Migrations**: Always run `npx prisma migrate deploy` (not `migrate dev`) against the Neon database. Run `migrate dev` only against your local database.
