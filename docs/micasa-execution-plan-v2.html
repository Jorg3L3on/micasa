Isolation is application-layer only: getOwnerContext() + ownerFilter on ~76 API routes, with no Postgres RLS backstop. One missing guard leaks a household’s full financial picture. Build a systematic cross-user test matrix before considering RLS (RLS is defense-in-depth, not a substitute for route tests).
Large Not started ↑ Data isolation
PRD: tasks/prd-cross-tenant-isolation.md GitHub: #108 open Notion: 1.3
Acceptance criteria · 0 / 4 core (+ 2 stretch)

PHASE 4 Ops & quality — know when things break ~2 weeks

Once real users are in play, you need visibility without opening Prisma Studio on a laptop. finance-log.ts helps but is not a substitute for error tracking or product analytics.

4.1 — Error monitoring (Sentry or equivalent)
No production error tracking today. Server exceptions, failed webhooks, and client crashes in the dashboard currently rely on users reporting bugs. Ship before or alongside Phase 2 public launch.
